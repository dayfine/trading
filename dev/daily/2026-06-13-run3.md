## Feature Progress

### cash-floor-correctness — NS1 closing-trade exemption (MERGED #1567)
The root fix for the #1553 −240% zombie. The live cash floor
(`Portfolio._check_sufficient_cash`) is an absolute-dollar solvency check that
rejected the THM short *cover* — the one trade that would have *reduced* risk —
stranding the position. NS1 adds a default-off
`exempt_closing_trades_from_cash_floor` flag that, when on, skips the floor for
the **reducing portion** of a closing trade (long sell / short cover), with the
over-cover flip exempting only the closing portion (`min(|trade_qty|,|pos_qty|)`
split mirroring `portfolio_margin.ml:_classify_trade`). Per
experiment-flag-discipline: R1 default-off (all goldens bit-equal), R2 axis-able
(pinned by `test_variant_matrix.ml`), R3 not wired on. A1 core-Portfolio change is
strategy-agnostic (qc-behavioral confirmed generalizability — long-sell-reducing
test proves it works beyond shorts). The author extracted `Portfolio_cash_floor` +
`Simulator_metrics` to stay under the 500-line file limit rather than bump it
(code-health-discipline honored).

### cash-floor-correctness — NS2 short-proceeds collateral design (MERGED #1569)
Read-only design-recommendation doc. Code-grounded root cause: margin-OFF
(`margin_config.ml:18`) short-circuits the collateral path, so a short `Sell` adds
positive proceeds to `current_cash` with `locked_collateral` stuck at 0, and

the apparent budget (over-deploys). Recommends **option (b)**: reserve proceeds as
locked collateral in the non-margin path behind a default-off flag + switch the

long-short track; run parallel, don't block. **Implementation is human-gated** —
