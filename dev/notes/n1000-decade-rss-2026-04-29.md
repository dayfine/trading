# N=1000×10y RSS measurement (2026-04-29)

First measured peak-RSS run on `goldens-broad/decade-2014-2023`, the
flagship tier-4 release-gate cell. Validates the engine-pool cost
model (PRs #618..#633) and the projection in
`dev/notes/panels-rss-matrix-post-engine-pool-2026-04-28.md`.

## Setup

- Build SHA: `ffd4f5d5` (post-#682, post-#683 sexp syntax fix).
- Container: `trading-1-dev`, opam env, **default GC** (no
  `OCAMLRUNPARAM` tuning).
- Scenario: `goldens-broad/decade-2014-2023.sexp`, universe_cap=1000,
  period 2014-01-02..2023-12-29 (10y, 2757 trading days, 1015 symbols
  including index + sector ETFs).
- Command:

  ```sh
  docker exec trading-1-dev bash -c \
    'cd /workspaces/trading-1/trading && eval $(opam env) && \
     /usr/bin/time -v _build/default/trading/backtest/scenarios/scenario_runner.exe \
       --dir /tmp/decade-cell --fixtures-root test_data/backtest_scenarios'
  ```

  (`/tmp/decade-cell` is a single-file scratch dir staging only the
  decade scenario so `scenario_runner` doesn't pull in the other
  goldens-broad cells.)

## Result

| Metric                       | Measured       | Cost-model projection (post-engine-pool, GC-untuned) |
|------------------------------|---------------:|-----------------------------------------------------:|
| Peak RSS                     | **2,945 MB**   | ~5,700 MB (β=3.94 MB/sym × 1000 + γ·N·(T−1) + α)    |
| Wall (real)                  | **4:26.65**    | not projected                                       |
| User CPU                     | 246.98 s       |                                                     |
| % CPU                        | 95 %           | single-threaded as expected                         |
| Major page faults            | 74             | minimal                                             |
| Minor page faults            | 694,118        |                                                     |
| Voluntary context switches   | 630,649        |                                                     |
| Exit status                  | 0              |                                                     |

The measured peak is **~52 % of the projection**. Likely sources of
the gap:

1. The projection in
   `panels-rss-matrix-post-engine-pool-2026-04-28.md` is calibrated on
   GC-untuned numbers but assumes GC tuning is applied, so the
   projected ceiling may bake in headroom.
2. The decade scenario's universe is broad-1000 (sectors.csv-derived,
   real symbols) vs the 292-symbol fixture used in the matrix —
   different per-symbol bar density / split-event load, which γ
   doesn't fully capture.
3. The split-day broker-model fixes (#678 / #680) added per-tick
   `_apply_splits_to_positions` and `Stops_split_runner.adjust` which
   trim some allocation churn elsewhere by short-circuiting stale
   state — net effect on RSS was probably mildly positive, not
   measured.

The shape (β coefficient stable, γ small, α negligible) is consistent
with the cost model. Lower absolute number is good news for capacity.

## Implications

- **Tier-4 release-gate is comfortably below the 8 GB ubuntu-latest
  ceiling** at N=1000×10y. Headroom ~5 GB.
- **Capacity envelope:** pushing N higher should be feasible up to
  ~N=2000–2500 within 8 GB before squeezing. Beyond that, the
  daily-snapshot streaming work (`dev/plans/daily-snapshot-streaming-2026-04-27.md`)
  remains the gate for N≥5000.
- **Wall:** 4:27 for a 10-year backtest at this scale is well within
  the tier-4 8h budget. Could probably batch-run all four
  goldens-broad cells in <30 min sequential.

## What this run does NOT establish

- **Trading correctness.** The decade scenario uses default config
  (shorts enabled). Metrics are blown by the same short-side bug
  documented in `dev/notes/short-side-gaps-2026-04-29.md` (return
  +1413 %, MaxDD 96 %). Real metrics await closure of G1-G4 or
  long-only override on this scenario.
- **Steady-state allocation churn.** No `--gc-trace` / `--memtrace` —
  the matrix at `panels-rss-matrix-post-engine-pool-2026-04-28.md`
  has the per-step allocation profile; this run just confirms peak.
- **GC-tuned peak.** With `OCAMLRUNPARAM=o=60,s=512k` the peak would
  likely drop further (per the matrix doc, GC tuning shaved
  ~25–37 % off untuned RSS for similar cells).

## Re-run trigger

Re-run when:

- A simulator-level memory regression is suspected (e.g. cliff in
  panels-rss-matrix-post-* or a new γ-affecting feature lands).
- The first long-only canonical baseline lands (likely after G1-G4 or
  the long-only override on goldens-broad). Re-pin BASELINE_PENDING
  ranges on `decade-2014-2023.sexp` against that run's metrics.
- A new universe size exceeds the projected envelope and we want to
  verify before relying on it.
