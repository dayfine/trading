The R2 rule statement ("trading/trading/weinstein/ must not import from analysis/weinstein/") is ambiguous with respect to `trading/trading/weinstein/strategy/`. The rule description explicitly mentions only `stops/` and `trading_state/` as the trading-side Weinstein modules. However, `trading/trading/weinstein/strategy/lib/dune` imports multiple analysis modules:
- `weinstein.snapshot_pipeline`
- `weinstein.snapshot_runtime`  
- `weinstein.stage`
- `weinstein.macro`
- `weinstein.rs`
- `weinstein.sector`
- `weinstein.screener`
- `weinstein.stock_analysis`

The architecture graph check does not flag this because it only detects `open` statements in .ml files, not dune-level dependencies. The strategy module is intended as an orchestration point between analysis and trading layers, and this appears to be intentional design. Recommend clarifying the rule statement in `docs/design/dependency-rules.md` to explicitly scope R2 to `stops/` and `trading_state/` only, not the entire `trading/trading/weinstein/` directory.
